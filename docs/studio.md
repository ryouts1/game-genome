# Studio

Studio は `games/*.json` を手で書く前の作業台です。完成品を commit する前に、ブラウザ上で編集、検証、試行、共有までできます。

## できること

- 既存ゲームやスターターテンプレートを editor に入れる
- JSON 構文エラーとルール定義エラーを分けて確認する
- Definition Health Check で完成度の弱点を見る
- プレビュー適用して board / AI / Arena に反映する
- LocalStorage に保存する
- スナップショットを残して編集途中を戻せるようにする
- JSON import / export を行う
- URL ハッシュ経由の共有リンクを作る

## Learned 候補の見方

局面分析を実行すると、Minimax / MCTS に加えて `Learned 候補` が出ます。ここでは、学習済みブックがその局面でどの手をどれだけ持っているかを確認できます。

- `Samples`: その手が自己対戦で何回選ばれたか
- `Value`: その手の平均結果
- `book hit`: その局面が学習済みブックに載っていたか
- `Genome AI が book を採用`: 閾値を超えていたので探索へ戻らず採用した状態

## Health Check の見方

- `Overall`: 全体の目安
- `Validity`: 必須項目や整合性が壊れていないか
- `Clarity`: 名前、記号、説明が整っているか
- `AI fit`: 探索や評価が扱いやすいか
- `Reuse`: 派生定義へ転用しやすいか

数値自体を絶対視するというより、README に書くときや、サンプルとして残すときの弱点探しに使う想定です。

## スナップショット

`ローカル保存` は catalog に残す操作、`スナップショット保存` は editor の途中版を退避する操作です。未完成な版を並行で試したいときは、まずスナップショットを使うほうが軽いです。
